# 冒险饰品系统

## 概述

冒险饰品系统是本模组的核心入口。两个 Curios body slot 饰品作为"钥匙"，数据存储在玩家 Capability 上，佩戴一次即永久激活对应状态（不可逆）。所有超模附魔通过 Capability 门禁判断是否可用。

## 两层饰品

| 饰品 | 触发效果 | 可逆性 |
|------|---------|--------|
| **冒险的开始** | 成为"冒险者"（adventurer），解锁基础超模附魔 + 开始追踪里程碑 | 不可逆 |
| **冒险的终点** | 永久完全解锁（fullyUnlocked），7 种高级被动能力可用 | 不可逆 |

## 三层状态模型

### 第一层：激活层（不可逆）

- `adventurer`：佩戴过"冒险的开始" → `true`，永久生效
- `fullyUnlocked`：佩戴过"冒险的终点" → `true`，永久生效（同时写计分板 `mme_adventure`）

### 第二层：里程碑层（仅 adventurer 时追踪）

5 个原版进度里程碑，按顺序解锁：

| 里程碑 | 触发条件 | 解锁内容 |
|--------|---------|---------|
| 下界 (NETHER) | 进入下界维度 | 闪避附魔 |
| 凋零 (WITHER) | 击杀凋零 | 时停斩附魔 |
| 监守者 (WARDEN) | 击杀监守者 | 至高附魔 |
| 末影龙 (DRAGON) | 击杀末影龙 | 淬魂附魔 |
| 鞘翅 (ELYTRA) | 获得鞘翅 | 允许合成冒险的终点（龙蛋 + 鞘翅 → 终点） |

### 第三层：查询层

所有门禁统一通过 `AdventureProgressCapability.getAdventureProgress(player)` 读取，返回 `IAdventureProgress` 接口。

```java
// 门禁检查模式
var progress = AdventureProgressCapability.getAdventureProgress(player);
if (progress.isEmpty()) return;  // 无 Capability，退出
if (!progress.get().isAdventurer()) return;  // 未激活冒险者
if (!progress.get().isNetherUnlocked()) return;  // 特定里程碑未解锁
```

## 9 种可开关能力

能力分两级：`adventurer` 级（佩戴"开始"即可）和 `fullyUnlocked` 级（需佩戴"终点"）。

### Adventurer 级（2 种）

| ID | 名称 | 效果 |
|----|------|------|
| `buff_extend` | 恩赐永驻 | 每 3 秒检查正面效果，低于保底值自动续期 |
| `double_jump` | 虚空踏步 | 空中可再次跳跃 |

### FullyUnlocked 级（7 种）

| ID | 名称 | 效果 |
|----|------|------|
| `death_defy` | 死亡抗拒 | 冷却制免死，触发后无敌 + 回满血 + 清负面 |
| `damage_resist` | 伤害抗性 | 全伤害固定比例减免（默认 25%） |
| `env_immunity` | 环境免疫 | 免疫火焰伤害 |
| `resilience` | 受击坚韧 | 每次受击叠层，每层额外减伤，5 秒无受伤重置 |
| `reject_health_manip` | 拒绝篡改 | 拦截外部 mod 对血量的非法修改 |
| `true_health` | 真实血量 | Capability 独立备份血量，自动修复被篡改的血量 |
| `active_skill` | 旅者之力 | 可使用主动技能（审判/庇护） |

## 数据持久化与同步

### 存储层级

1. **Capability**（`IAdventureProgress`）：服务端权威数据源
2. **PersistentData**（`MME_AdventureProgress` 键）：Entity NBT，每次写入 Capability 时同步，跨维度/死亡可恢复
3. **物品 NBT**（`MME_Milestone_*` 键）：只读缓存，用于客户端显示
4. **计分板**（`mme_adventure`）：fullyUnlocked 标记

### 恢复策略

```
onPlayerClone (死亡/跨维度) →
  1. 读 persistentData
  2. 尝试旧 Capability（reviveCaps）
  3. 扫描全身物品 NBT
```

### 客户端同步

- 登录/维度切换/里程碑解锁时推送 `AdventureSyncPacket`
- 客户端收到后更新本地 Capability 缓存
- 能力面板和 tooltip 依赖客户端 Capability 数据

## 能力开关

所有能力可通过 P 键打开的"冒险之力"面板独立开关。切换后立即生效：

- `true_health` 关闭时自动重置备份血量（防止过期备份误判）
- `buff_extend` 关闭时清除 Buff 检查计时器
- 能力开关状态存储在 `disabledAbilities` 集合中（Capability → NBT 持久化）
