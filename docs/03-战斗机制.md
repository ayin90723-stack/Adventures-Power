# 战斗机制

## 见既斩 — 穿透一切无敌

### 设计目标
攻击可以穿透目标的一切无敌状态（包括其他 mod 自定义的无敌帧、Boss 阶段无敌、格挡系统）。

### 四层穿透架构

```
Layer 0: PlayerAttackMixin
  ├─ 注入点: Player.attack() 内 target.hurt() @Redirect
  ├─ 问题: 部分 Boss 重写 hurt() 完全绕过 super.hurt()
  └─ 方案: 在调用前用 InvulClearUtil 清除目标自定义无敌字段

Layer 1: SeeAndSlashMixin
  ├─ 注入点: Entity.isInvulnerableTo() HEAD
  └─ 方案: 持有者直接返回 false，穿透一切无敌判断

Layer 2: SeeAndSlashLivingEntityMixin — hurt() 拦截
  ├─ 注入点: LivingEntity.hurt() 内 ForgeHooks.onLivingHurt() @Redirect
  ├─ 问题: 部分 mod 在 onLivingHurt 中取消伤害
  └─ 方案: Redirect 绕过 ForgeHooks.onLivingHurt 调用

Layer 2.5: SeeAndSlashLivingEntityMixin — 限伤单向
  ├─ 问题: 部分 Boss 将 receivedDamage 钳制为 0，使 hurt 内部逻辑短路
  └─ 方案: Inject 在 hurt() 开头限定 damage >= 1.0F，单向不可降

Layer 3: SeeAndSlashLivingEntityMixin — 兜底穿透
  ├─ 注入点: hurt() RETURN + actuallyHurt Invoker
  ├─ 问题: hurt() 返回 false（伤害被取消）但实际未造成伤害
  └─ 方案: 返回 false 时直接调用 actuallyHurt（原版内部伤害逻辑），绕过一切拦截
```

### 门禁
需要 `fullyUnlocked`（佩戴冒险的终点）。

### 辅助工具

- **InvulClearUtil**：反射遍历实体继承链，清除命名匹配的无敌字段（int/boolean 型），支持 Goety Apostle、Mowzie 钢铁守护者等。
- **HealthUtil**：反射写入 SynchedEntityData 血量字段，绕过 setHealth() 覆写链。

## 不死斩 — 禁疗与反复活

### 效果
攻击目标施加"不死斩印记"Buff，持续期间：
- 禁止目标回血（tick 末钳制血量到追踪的低点）
- 禁止目标复活（UndyingSlashEffect 拦截）

### 实现

```
UndyingSlashEffect（Buff）
├─ onApply: 记录当前血量作为"低点"
├─ onTick: 持续更新低点（血量下降时），钳制回低点（血量回升时）
└─ onExpire: 自动清理

UndyingSlashMixin
├─ setHealth HEAD: 拦截 setHealth 调用，不是来自 hurt 路径的回血 → 钳制
├─ tick TAIL: 每 tick 末再次检查，用反射直写强制钳制（反制 DataItem 篡改）
└─ 钳制公式: health = Math.min(current, clampedLow)
```

### 与淬魂的联动
有不死斩标记的目标受到淬魂伤害时，额外 ×1.5 倍率。

## 淬魂 — 多维伤害加成

### 与原版的区别
**完全重写。** 原版淬魂是简单线性伤害加成，当前版本改为多维计算：

```java
总伤害 = 基础固伤 × 等级
       + 目标最大生命值 × 0.5% × 等级
       + 目标当前生命值 × 0.5% × 等级
       × (有不死斩标记 ? 1.5 : 1.0)
```

### 门禁
需要末影龙里程碑（`isDragonUnlocked()`）。

## 淬魂Plus — 影子血量独立斩杀

### 设计目标
在目标 NBT（`getPersistentData()`）中维护独立的"影子血量"，不经过标准伤害路径。影子血量归零时触发秒杀，无视一切防御。

### 效果
- 每次攻击：固定伤害 + 目标最大生命值百分比 → 从影子血量扣除
- Buff 持续期间影子血量有效，到期自动过期
- 影子血量 ≤ 0 时触发斩杀

### 三层斩杀路径

```java
1. hurt()   → 影子血量归零 → 直接调用 target.hurt(∞)
2. die()    → hurt() 未杀死 → LivingDeathEvent 中调用 target.die()
3. saturationKill() → die() 仍失败 → 饱和式斩杀（clearFire + kill + setHealth(0) + die 循环）
```

三层均无条件执行，确保即使前一层被外部 mod 拦截，后续层仍能完成斩杀。

### 对玩家无效
仅对非玩家实体生效。

### 影子血量显示
目标头顶显示 BossBar 风格的影子血量条。

### 门禁
需要 `fullyUnlocked`（佩戴冒险的终点）。

---

## 闪避 — 概率免疫伤害

- 基础概率：0%（1 级从 4% 开始）
- 每级 +4%
- 触发时完全免疫该次伤害 + 粒子特效
- **门禁**：需要下界里程碑

---

## 闪避与伤害抗性的区别

| 机制 | 闪避 (Dodge) | 伤害抗性 (Damage Resist) |
|------|-------------|------------------------|
| 来源 | 附魔 | 冒险能力 |
| 效果 | 概率完全免疫 | 固定比例减伤 |
| 门禁 | 下界里程碑 | FullyUnlocked |
