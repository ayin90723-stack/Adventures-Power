# 新模组架构设计

## 命名方案（暂定）

| 项目 | 当前 | 新模组（建议） |
|------|------|--------------|
| 模组名 | MyMoreEnchantments | Adventure's Power（冒险的力量） |
| Mod ID | `mme` | `adventure_power` 或 `advpower` |
| 包名 | `com.main.mymoreenchantments` | `com.rosycentury.adventurepower` |

## 保留范围

### 附魔（8 个）
见既斩、不死斩、淬魂、淬魂Plus、闪避、永恒、传承、不离不弃

### 物品（2 个）
冒险的开始、冒险的终点（其他全部不留）

### 系统（全部保留）
Mixin（11 个）、Capability、血量安全、冒险能力（9 种）、主动技能（2 种）、UI、配置

## 建议包结构

```
com.rosycentury.adventurepower/
├── enchantment/       # 8 个附魔元数据类
├── handler/           # EnchantmentHandler（精简为 8 个 handle*()）
├── ability/           # 9 种冒险能力
├── skill/             # 2 种主动技能
├── capability/        # Forge Capability
├── mixin/             # 11 个 Mixin
├── item/              # 仅 2 个饰品 + 配方
├── effect/            # UndyingSlashEffect 等
├── config/            # 精简配置
├── ui/                # 3 面板 + 2 HUD
├── network/           # 网络包
├── util/              # HealthUtil / InvulClearUtil 等
└── compat/            # 兼容层
```

## 迁移优先级

### 第一阶段：核心基础设施
1. Gradle 项目搭建（Forge MDK + Mixin + Curios）
2. Mixin 配置文件 + refmap
3. Capability 系统
4. 配置系统（只保留 8 附魔 + 冒险能力）
5. ModItems（仅 2 个饰品）

### 第二阶段：战斗核心
6. 见既斩（4 层 Mixin + InvulClearUtil + HealthUtil）
7. 不死斩（Buff + Mixin）
8. 淬魂（重写伤害公式）
9. 淬魂Plus（影子血量 + 三层斩杀）
10. 闪避（拦截整个 hurt）

### 第三阶段：血量安全
11. 真实血量 Mixin
12. 拒绝篡改 × 2
13. 死亡抗拒（事件 + Mixin）

### 第四阶段：能力与 UI
14. 冒险饰品里程碑事件
15. 9 种能力实现
16. Buff 管理 + 冒险之力面板
17. 主动技能 + HUD

### 第五阶段：附魔整合
18. 8 个附魔注册 + 元数据类
19. EnchantmentHandler 精简
20. 永恒/传承/不离不弃 特殊逻辑

### 第六阶段：打磨
21. 语言文件
22. 纹理资源（冒险开始/终点）
23. 伤害类型注册 + 合成配方
24. 兼容性测试

## 不迁移的内容

- 所有原作者附魔（时停斩、至高、生命汲取、暴击... 全部 40+ 个）
- 所有原作者物品（超越晶剑、下界绝魔刀、地狱图腾... 全部）
- 所有生物交互机制（唤魔者、女巫、嗅探兽... 全部）
- 爱弦之诗系列
